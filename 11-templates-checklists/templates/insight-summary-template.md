# Insight Summary Template

Use this template when writing an analytical finding from a report, dataset, dashboard, model, or sandbox experiment.

This is especially useful for data analysts, report developers, data scientists, department representatives, and Fabric enthusiasts.

The purpose of this template is to help learners write observations carefully, separate evidence from interpretation, and avoid overclaiming.

## 1. Analytical Question

| Field                          | Response                                                    |
| ------------------------------ | ----------------------------------------------------------- |
| Question being explored        |                                                             |
| Dataset, report, or model used |                                                             |
| Workspace                      |                                                             |
| Workspace type                 | Personal / Sandbox / Department / BIA Production            |
| Output status                  | Sandbox / Draft / Prototype / UAT / Department / Production |
| Analysis date                  |                                                             |
| Analyst or learner             |                                                             |

## 2. Observation

Write the main observation clearly.

```text
The analysis suggests that...
```

## 3. Evidence

Summarise the evidence supporting the observation.

| Evidence Area                               | Response |
| ------------------------------------------- | -------- |
| Metric, visual, model output, or table used |          |
| Filter or segment applied                   |          |
| Time period considered                      |          |
| Comparison group, if any                    |          |
| Key value or pattern observed               |          |
| Data source or report page used             |          |

## 4. Why It Matters

Explain why the finding may be useful.

```text
This may be useful because...
```

## 5. Caveat

State what users should be careful about.

```text
This should be interpreted carefully because...
```

Examples of caveats may include:

* The data may not be current
* Filters or slicers may affect the view
* The analysis is descriptive and does not prove causality
* Some important variables may not be included
* The output is based on sandbox or experimental work
* The metric definition may need validation

## 6. Possible Misinterpretation

Identify what someone might wrongly conclude.

```text
A possible misinterpretation is...
```

## 7. Follow-Up Question

Identify the next question to explore.

```text
The next question to explore is...
```

## 8. Additional Data Needed

List any data that would strengthen the analysis.

```text
Additional data that may strengthen the analysis:
```

## 9. Recommended Next Step

| Option                                            | Select One |
| ------------------------------------------------- | ---------- |
| Keep as sandbox learning only                     |            |
| Explore further                                   |            |
| Validate with data owner or subject matter expert |            |
| Discuss as possible department use case           |            |
| Escalate to BIA for review                        |            |

## 10. Sharing Reminder

Before sharing this insight more widely, check:

* [ ] The output status is clear
* [ ] The data sensitivity is understood
* [ ] The caveats are included
* [ ] The insight is not presented as production-ready unless it has been reviewed
* [ ] Sandbox findings are clearly labelled as learning or experimental
* [ ] Non-BIA users consume approved production outputs through approved report or app sharing channels, where applicable

## 11. Final Summary

Write a short, plain-language summary.

```text
In summary...
```
