# Use Case Canvas

Use this template when framing a department analytics use case before moving beyond sandbox or requesting BIA support.

This is especially useful for department representatives, workspace owners, analysts, and project leads.

The purpose of this canvas is to clarify the business question, intended users, data needs, ownership, access considerations, operational needs, and recommended next step.

## 7. Workspace Boundary Review

| Workspace Type           | Use Case Fit                                                                                                           |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------- |
| Personal Workspace       | Suitable only for private drafts and individual exploration. Not suitable for department use cases or shared assets.   |
| Sandbox Workspace        | Suitable for learning, experimentation, and early exploration using safe data. Not suitable for official reporting.    |
| Department Workspace     | Suitable for approved department-level exploration, prototyping, and working assets with clear owner and deputy owner. |
| BIA Production Workspace | Suitable only for BIA-managed production assets, with direct workspace access restricted to BIA users.                 |

Confirm:

* [ ] The proposed workspace type matches the maturity and purpose of the use case
* [ ] The use case is not being kept in a personal workspace if it needs department ownership
* [ ] Sandbox work is not being treated as official or production-ready
* [ ] If a department workspace is needed, both workspace owner and deputy workspace owner are identified
* [ ] If the use case may become production-facing, BIA review is requested
* [ ] If the use case becomes a BIA production output, non-BIA users will consume it through approved report or app sharing channels, not direct BIA Production Workspace membership
