# Report Context Note

Use this template when reviewing a report before interpreting, sharing, or relying on it.

This is especially useful for report consumers, report developers, department representatives, and workspace owners.

The purpose is to help users check the report context before drawing conclusions from the visuals.

## 3. Workspace and Sharing Reminder

Check the workspace context before interpreting or sharing the report.

| Workspace Type           | Reminder                                                                                                             |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------- |
| Personal Workspace       | Reports are private drafts or individual exploration only. They should not be treated as shared or official outputs. |
| Sandbox Workspace        | Reports are learning artefacts. They should not be treated as official or production-ready.                          |
| Department Workspace     | Reports may support department work, but ownership, access, sensitivity, and validation should be clear.             |
| BIA Production Workspace | Reports are BIA-managed production assets. Direct workspace access is restricted to BIA users.                       |

Confirm:

* [ ] I understand whether this report is experimental, department-facing, or production-facing
* [ ] I understand that sandbox reports are not official production reports
* [ ] I understand that workspace access does not automatically allow export, reuse, redistribution, or onward sharing
* [ ] I understand that non-BIA users should consume approved production outputs through approved report or app sharing channels, where applicable
