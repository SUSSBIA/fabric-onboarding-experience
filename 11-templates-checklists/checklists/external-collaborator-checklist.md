# External Collaborator Checklist

Use this checklist before granting access to an external collaborator.

External collaborators may require additional review because they are not University staff and may need Microsoft Entra B2B guest access, licensing review, time-bound access, and stronger data sensitivity checks.

## 2. Purpose of Access

- [ ] The purpose of access is clearly documented
- [ ] The collaborator’s role is clear
- [ ] The work is linked to an approved project, use case, or engagement
- [ ] The expected output is clear
- [ ] The access requested is necessary for the stated purpose
- [ ] The access is not broader than needed

## 4. Microsoft Entra B2B Guest Access

- [ ] Microsoft Entra B2B guest account requirement has been reviewed
- [ ] Guest account creation process is understood
- [ ] Guest identity is linked to the correct external email
- [ ] Access will be governed through the approved University identity process
- [ ] Guest access removal plan is identified
- [ ] Owner for guest access review or removal is identified

## 5. Licensing Review

- [ ] Power BI Pro or Premium Per User requirement has been reviewed
- [ ] It is clear whether the collaborator can use their own organisation’s license
- [ ] It is clear whether the University needs to assign a license
- [ ] Viewing-only scenario has been reviewed separately from editing or publishing
- [ ] Licensing requirement is confirmed before access is granted
- [ ] BIA or relevant administrator review is requested where needed

## 6. Data Sensitivity Review

- [ ] Data sensitivity has been assessed
- [ ] Sensitivity label is known or checked
- [ ] The collaborator is authorised to access the data
- [ ] Confidential or restricted data exposure has been reviewed
- [ ] Export and screenshot risks have been considered
- [ ] Sharing beyond the intended collaborator is not allowed unless approved
- [ ] The collaborator understands handling expectations

## 7. Access Scope and Least Privilege

- [ ] Access is limited to the minimum required workspace or item
- [ ] Viewer access is used where editing is not required
- [ ] Contributor or Member access is justified, if requested
- [ ] Admin access is avoided unless there is a specific approved reason
- [ ] Build permission is only granted if needed
- [ ] Access is time-bound where possible
- [ ] Review date is recorded

## 8. Security and Governance

- [ ] External sharing policy has been considered
- [ ] Tenant-level settings are not bypassed
- [ ] Workspace role and item permission interaction is understood
- [ ] Row-Level Security is reviewed, if applicable
- [ ] The collaborator cannot access unintended content
- [ ] Any contractual, procurement, or vendor-related constraints are considered
- [ ] BIA review is completed where required
